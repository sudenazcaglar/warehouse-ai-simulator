## Index and Unique Constraint Verification

A difference was observed between the SQLAlchemy model metadata index counts and the PostgreSQL schema inspection index counts for some tables.

This is expected because SQLAlchemy `table.indexes` reports only explicitly declared `Index(...)` objects, while PostgreSQL also creates physical indexes automatically for `UniqueConstraint(...)` definitions.

The following unique constraints were verified in PostgreSQL:

```text
environment_configs | uq_environment_configs_name_version | UNIQUE (name, version)
agents              | uq_agents_run_agent_name            | UNIQUE (simulation_run_id, agent_name)
checkpoints         | uq_checkpoints_session_step         | UNIQUE (training_session_id, step)
episodes            | uq_episodes_run_episode_number      | UNIQUE (simulation_run_id, episode_number)
model_versions      | uq_model_versions_name_version      | UNIQUE (model_name, version)
```

The observed index count difference is expected and does not indicate a schema or migration problem.
